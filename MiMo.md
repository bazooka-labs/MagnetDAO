## What Was Built

### Smart Contracts (Algorand / PyTeal)

**Governance Contract** (`contracts/governance/governance.py`)
- Quarterly proposal lifecycle: create → discuss → vote → finalize
- On-chain voting with 1 Magnet = 1 Vote weighting
- Founder retains final approval authority over all liquidity decisions
- Box-based proposal storage with fields: project name, liquidity pair, capital requested, timeline, risk hash
- Spam prevention via 1 Algo proposal deposit
- Functions: `create_proposal`, `open_voting`, `cast_vote`, `close_voting`, `finalize`, `advance_quarter`, `update_founder`

**Treasury Contract** (`contracts/treasury/treasury.py`)
- Quarterly funding deposits from Bazooka Labs revenue
- Deployment tracking: create → execute → withdraw
- Inner transaction execution for fund transfers
- Fee collection from swap pools
- Founder-controlled withdrawals and authority updates
- Functions: `deposit`, `create_deploy`, `execute_deploy`, `receive_fees`, `withdraw_fees`, `update_founder`, `update_gov_app`

**Deployment Script** (`contracts/deploy.py`)
- Compiles both contracts via Algorand's `compile` endpoint
- Supports `--network testnet|mainnet` and `--magnet-asa` flags
- Outputs `deployment.json` with app IDs post-deploy

### Web Application (Next.js 14 + TypeScript + Tailwind CSS)

**Pages:**

| Route | Description |
|---|---|
| `/` | Landing page — hero, token stats (ASA 3081853135, 750K supply), 4-step how-it-works flow, feature cards, CTA |
| `/proposals` | Proposal list with all/active/completed filters, expandable cards with vote progress bars, proposal submission form, vote for/against buttons (shown when wallet connected and proposal is in voting phase) |
| `/treasury` | Dashboard with 6 stat cards (total funded, deployed, balance, active pools, fees, token), deployment list with status badges, quarterly summary table, chart placeholder for indexer data |
| `/governance` | Quarterly cycle explainer (submit → discuss → vote → deploy), voting rules breakdown, founder authority section, no-proposals fallback (rollover / founder nomination) |

**Components:**
- `Navbar` — Logo, nav links, wallet connect/disconnect button, mobile hamburger menu
- `Footer` — GitHub, Discord, AlgoExplorer links
- `StatCard` — Reusable metric display with label, value, sublabel, icon
- `StatusBadge` — Color-coded pill for Pending/Voting/Approved/Rejected/Deployed/Active/Withdrawn
- `Card`, `SectionHeader`, `EmptyState` — Shared layout primitives

**Infrastructure:**
- `useWallet.tsx` — Pera Wallet Connect hook with session reconnect
- `algorand.ts` — AlgoSDK client helpers (algod, indexer, state parsing, formatting)
- `constants.ts` — ALGOD/INDEXER URLs, token metadata, quarter duration
- `types/dao.ts` — Full TypeScript types for Proposal, Deployment, Vote, TreasuryState, GovernanceState, DaoConfig

**Deployment:**
- Live on Vercel at https://magnetdao.vercel.app
- Project: `bazooka-creates/magnetdao`
